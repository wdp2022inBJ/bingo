# 运动启动 Bingo：腾讯云 COS 部署说明

## 发布包内容

```text
运动启动Bingo-腾讯云发布包/
├── index.html
└── assets/
    └── bingo/
        ├── action-01.jpg
        ├── ...
        └── action-25.jpg
```

必须同时上传 `index.html` 和整个 `assets` 文件夹，否则第二关的动作图片无法显示。

## 方法一：使用腾讯云控制台上传

1. 登录腾讯云控制台，进入「对象存储 COS」。
2. 创建一个存储桶，建议选择距离访问用户较近的地域。
3. 将访问权限设置为「公有读私有写」，或按实际安全需求配置公开读取。
4. 打开存储桶，上传发布包里面的全部内容：
   - `index.html` 上传到存储桶根目录。
   - `assets` 文件夹也上传到存储桶根目录。
5. 进入「基础配置 > 静态网站」：
   - 开启静态网站。
   - 索引文档填写：`index.html`
   - 强制 HTTPS：开启
6. 使用 COS 提供的静态网站访问地址打开页面。

注意：上传的是发布包里面的文件，而不是把发布包文件夹作为最外层目录上传。访问地址根目录下必须直接存在 `index.html`。

## 方法二：使用 COSCMD 命令上传

### 1. 安装 COSCMD

在 PowerShell 中执行：

```powershell
pip install coscmd
```

### 2. 配置 COSCMD

从腾讯云「访问管理 > API 密钥管理」获取 SecretId 和 SecretKey。

替换下面四项后执行：

```powershell
coscmd config -a 你的SecretId -s 你的SecretKey -b 你的存储桶名称-APPID -r 你的地域
```

示例地域：

```text
ap-shanghai
ap-guangzhou
ap-beijing
```

不要把 SecretId 和 SecretKey 发到聊天、笔记或公开仓库。

### 3. 上传完整发布包

在 PowerShell 中执行：

```powershell
coscmd upload -r "C:\Users\MAGIC\Documents\Codex\2026-05-29\new-chat\运动启动Bingo-腾讯云发布包\" /
```

更新页面后，再次执行相同命令即可覆盖线上文件。

## 访问地址

开启 COS 静态网站后，使用控制台显示的静态网站域名访问。常见格式类似：

```text
https://你的存储桶名称.cos-website.你的地域.myqcloud.com
```

实际地址以腾讯云 COS 控制台显示结果为准。

## 部署后检查

- 首页可以正常打开。
- Bingo 1 翻卡后能显示萌宠。
- Bingo 2 翻卡后能显示动作图片。
- 手机端 5×5 棋盘未发生横向溢出。
- 刷新页面后闯关进度仍然保留。

闯关进度保存在访问者浏览器的 `localStorage` 中，不会上传到腾讯云，也不会跨浏览器同步。
